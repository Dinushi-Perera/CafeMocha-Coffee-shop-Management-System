Caf√© Mocha, a popular coffee shop frequented by thousands of customers each month, currently relies on a manual system for managing orders, billing, and record-keeping, which has proven inefficient and error-prone. This project seeks to develop a computerized system that automates these operations, enhancing accuracy, efficiency, and overall customer satisfaction. The system aims to provide a user-friendly interface for managing customer orders, billing, and records, with features including a secure login system, menu item management, accurate bill generation with taxes and discounts, and a help feature for user guidance. The scope includes a console-based Java application with data stored in text files, addressing the inefficiencies of the manual system and improving service speed and reliability. It assumes users are familiar with basic computer operations, the system will be used in a single location, and all transactions are processed in real-time.
<br/> System Requirements <br/>
2.1	Functional Requirements
<br/>
User Authentication: Login using a username and password.
Order Management: Ability to add, edit, and delete customer orders.
Bill Calculation: Automatically calculate bills with applicable taxes and discounts.
Menu Management: Add, update, or remove menu items.
Help Functionality: Provide guidance to users on how to use the system.
Exit Functionality: Allow users to properly log out and close the system.
<br/>
2.2	Non-Functional Requirements
<br/>
Usability: The system should be easy to navigate and use.
Reliability: Ensure accurate data processing and storage.
Performance: The system should respond quickly to user inputs.
Security: Protect user data with secure login and encrypted data storage.
<br/>
2.3	Software Requirements
<br/>
Software: Java Development Kit (JDK), Text Editor or IDE.
<br/>
2.4	Constraints
<br/>
The system will be console-based and will not use a graphical user interface.
Data will be stored in plain text files, not in a database.
